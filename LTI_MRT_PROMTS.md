# Prompt para obtención de user storyes
Detecta los tipos de usuarios de la aplicación. Con estos tipos de usuario y sus user-cases genera user-storys
Para cada user-story crea un título incluyendo usando el formato "Como [rol de usuario], quiero [objetivo] para que pueda [beneficio]". También incluye una descripción.
Las user-stroy debe estar enfocada al usuario, mantener un formato simple.
Priorizalas usando el criterio de aportar un MVP en pocos sprints
